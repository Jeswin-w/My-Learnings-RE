## What is a Lambda?

A lambda is an anonymous function that can be:
- Stored in a variable
- Passed as an argument
- Returned from another function

```kotlin
val sum = { a: Int, b: Int -> a + b }
```

---

## Lambda Syntax

```kotlin
{ parameters -> body }
```

---

## Type Inference

```kotlin
val double: (Int) -> Int = { it * 2 }
```

---

## The `it` Keyword

```kotlin
listOf(1, 2, 3).map { it * 2 }
```

---

## Multiple Parameters

```kotlin
val multiply = { a: Int, b: Int -> a * b }
```

---

## Lambdas with Multiple Statements

```kotlin
val compute = { x: Int ->
    val y = x + 1
    y * 2
}
```

---

## Lambdas as Function Arguments

```kotlin
fun operate(x: Int, y: Int, op: (Int, Int) -> Int): Int {
    return op(x, y)
}

operate(2, 3) { a, b -> a + b }
```

---

## Trailing Lambda Syntax

```kotlin
operate(2, 3) { a, b -> a * b }
```

---

## Anonymous Functions vs Lambdas

Anonymous function:
```kotlin
fun(x: Int): Int {
    return x * 2
}
```

Lambda:
```kotlin
{ x: Int -> x * 2 }
```

---

## Function References

```kotlin
fun square(x: Int) = x * x

listOf(1, 2, 3).map(::square)
```

---

## Common Mistakes

- Overusing `it`
- Very long lambdas
- Poor readability

---

## Key Takeaways

- Lambdas are concise
- Trailing lambdas improve readability
- Prefer clarity over brevity
